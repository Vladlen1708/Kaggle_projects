# Titanic — Machine Learning from Disaster

Предсказание, кто из пассажиров «Титаника» выжил в катастрофе, по их характеристикам (класс каюты, пол, возраст и т.д.). Классическое вводное соревнование Kaggle.

**Kaggle:** [Titanic - Machine Learning from Disaster](https://www.kaggle.com/competitions/titanic)
**Тип задачи:** бинарная классификация (`Survived`: 0 / 1)
**Метрика:** Accuracy (доля верно предсказанных пассажиров)
**Результат:** 0.76315

---

## Данные

- `train.csv` — 891 пассажир (с известным `Survived`)
- `test.csv` — 418 пассажиров
- Целевая переменная: `Survived` (1 — выжил, 0 — нет)

Признаки:

| Признак | Описание |
|---|---|
| `Pclass` | Класс билета (1 / 2 / 3) — маркер социально-экономического статуса |
| `Sex` | Пол |
| `Age` | Возраст (есть пропуски) |
| `SibSp` | Число братьев/сестёр/супругов на борту |
| `Parch` | Число родителей/детей на борту |
| `Fare` | Стоимость билета |
| `Cabin` | Каюта (много пропусков) |
| `Embarked` | Порт посадки (C / Q / S) |
| `Ticket`, `Name` | Номер билета и имя (можно извлечь титул) |


## EDA - Ключевые находки

- [Пассажиры 1 класса выживали чаще, чем 3 класса]

## Feature engineering

- Пропуски `Age`: медиана
- Пропуски `Embarked`, `Fare`: most_frequent
- `Title` из `Name` (Mr, Mrs, Miss, Master ...)
- `FamilySize` = `SibSp` + `Parch` + 1; признак `IsAlone`
- Кодирование категориальных: one-hot encoding

## Модели и валидация

| Модель | Валидация | CV score | LB score |
|---|---|---|---|
| XGBoost | 5-fold CV  | 0.84 | 0.763 |
| Catboost | 5-fold CV | 0.81 | — |
| Logistic Regression | 5-fold CV | 0.82 | 0.768 |

- Схема валидации: K-fold
- Подбор гиперпараметров: вручную (GridSearch начал уводить в переобучение)

## Результат

Имеется разрыв между CV (0.84) и LeaderBorad (0.76), это может говорить о небольшом переобучении
GridSearch лучшее качество на train с 500 деревьями, но бустингу столько не нужно, поэтому я эксперементровал в районе 300.
Добавил регуляризацию, но она не особо помогла. 

## Выводы и что улучшить

Возможно стоит протестировать еще Catboost, потому что в датасете много категориальных признаков, а так же попробовть Logistic Regression <br>
